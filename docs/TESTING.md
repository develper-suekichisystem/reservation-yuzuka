# 予約システム 動作テストチェックリスト

対象リリース：**1枠1予約・1日2人上限・予約間1時間休憩・妖精シルエット**

## 前提・事前準備

- [x] マイグレーション `supabase/migrations/20260702_available_slots_max_bookings_1.sql` を Supabase に適用済み
- [x] Vercel の環境変数を確認
  - [x] `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY`
  - [x] `LINE_CHANNEL_ACCESS_TOKEN`（LINE通知）
  - [x] `ADMIN_LINE_USER_ID`（管理者通知が必要な場合）
  - [x] `CRON_SECRET`（keep-alive cron 用）
- [x] テスト用に、ある1日を **管理画面（`#admin` → 受付設定）で 9:00〜18:00 など連続した時間帯を開放**しておく
- [x] 鑑定枠は「見かけ90分／実枠2時間」設定（既定メニュー）であることを確認

> メモ：以下の時間は「実枠2時間＋前後1時間休憩」を前提にした例です。開放した時間帯に合わせて読み替えてください。

---

## ① 1枠1予約・1日2人上限

- [x] **1件目を予約**（例：13:00）→ 予約完了画面まで進める
- [x] 同じ日をもう一度開くと、**13:00〜14:00 が × 表示**になっている（1枠1予約）
- [x] **2件目を予約**（例：16:00 など、下記③の休憩を満たす時間）→ 完了できる
- [x] 2件入った後に同じ日を開くと、**その日の全時間が × 表示**になる（1日2人上限）
- [x] 別の日を開くと、通常どおり予約できる（上限は日ごとにリセット）

## ② 見かけ90分／実枠2時間

- [x] 時間選択・確認画面の所要時間表示が **90分ベース**になっている
- [x] 1件予約すると、内部的に **2時間分**が押さえられている
  （13:00予約後、14:00 も選べなくなることで確認）

## ③ 予約の間に1時間の休憩

- [x] 13:00 に1件予約した状態で同じ日を開く
- [x] **15:00 が休憩で × 表示**になっている（枠13〜15時＋休憩15〜16時）
- [x] **16:00 は ○ で予約できる**
- [x] 逆順の確認：先に **16:00 を予約** → その後 **15:00 や 14:00 が × 表示**になる
  （どちらの順でも予約間は必ず1時間空く）

## ④ タンポポ妖精シルエット

- [ ] 読み込み中（ローディング）の**中央マークが妖精シルエット**になっている
- [ ] 拡大・高解像度端末でも輪郭がぼやけない（SVG）
- [ ] フワッと浮遊するアニメーションが動作している
- [ ] 色みがテーマ（イエロー→桜ピンク）に合っている

## LINE通知（回帰確認）

- [ ] 予約完了後、**お客様のLINEに確認メッセージ**が届く
- [ ] `ADMIN_LINE_USER_ID` 設定時、**管理者にも通知**が届く
- [ ] メッセージ内の日時・場所・予約番号が正しい

## 場所・オンライン/対面（回帰確認）

- [x] オンライン選択 → 場所選択なしで進める
- [x] 対面選択 → 登録済みの場所から選択できる
- [x] 確認画面・通知に選んだ鑑定方法が反映される

## 管理画面（回帰確認）

- [x] `#admin` でログインできる
- [x] 受付設定で日付ごとに時間帯を開放／削除できる
- [x] 予約一覧に、テストで入れた予約が表示される
- [x] 予約のキャンセルができ、キャンセル後はその枠が再び ○ になる

## 異常系・境界

- [x] 開放時間が1時間しかない日は、実枠2時間を確保できず ○ が出ない（想定どおり）
- [x] 同時に2人が同じ枠を予約しようとした場合、後発は「満枠」エラーになる
- [x] 過去日・未開放日は選択できない
